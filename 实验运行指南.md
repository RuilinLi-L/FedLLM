# DAGER 实验运行指南

本文汇总主仓库 [README.md](README.md) 与防御 baseline 说明 [DEFENSE_BASELINES.md](DEFENSE_BASELINES.md) 中的实验入口：**环境准备、主论文实验、联邦学习、消融、微调、防御实验**；每条命令后附参数含义。

**通用约定：** 以下命令均在项目根目录 `dager-gradient-inversion/` 下执行（`cd` 到该目录后再运行）。脚本路径以 `./scripts/...` 为准。

---

## 1. 环境与数据

### 1.1 Conda 环境

| 步骤 | 命令 |
|------|------|
| 创建环境 | `conda env create -f environment.yml -n dager` |
| 激活 | `conda activate dager` |

### 1.2 目录与模型

```bash
mkdir -p models models_cache
```

从 [Megadrive](https://mega.nz/folder/6E0ilJRA#ALIOb7a8ugfKOSmhCSEW6A) 下载微调 GPT-2、LoRA 等权重，放入 `models/`。

### 1.3 HuggingFace（可选）

- **ECHR 长序列数据集：** 按 README「Setting up HuggingFace」下载 CSV 并放到 `models_cache/datasets--glnmario--ECHR/`。
- **LLaMA-2：** 需 HF 账号与许可；`export HF_TOKEN=<你的 token>`，必要时 `huggingface-cli download <model> --cache-dir ./models_cache`。

### 1.4 Rank 容差（经验值）

| 批量大小 | 建议 `RANK_TOL` 量级 |
|----------|----------------------|
| 1 / 2 | `1e-7` |
| 4–16 | `1e-8` |
| 32–128 | `1e-9` |

不传 `--rank_tol` 时由实现自动推断（见 NumPy 文档语义）。

## 2. 终端日志（`scripts/` 与 `train.py`）

运行 `scripts/` 下辅助脚本时，**默认**把完整终端输出保存到 `log/runs/`，终端会打印 `[dager] Terminal log: ...`。

| 环境变量 | 作用 |
|----------|------|
| `DAGER_LOG_DIR=/path/to/dir` | 指定日志根目录 |
| `DAGER_NO_AUTO_LOG=1` | 关闭自动写文件日志 |

`python3 train.py ...` 默认生成 `log/runs/train_<dataset>_<timestamp>.txt`；可用 `--log_file PATH` 指定路径，或 `DAGER_NO_AUTO_LOG=1` 跳过。

## 3. 主实验：DAGER 攻击（论文表 1、2、3、5、11）

底层均为调用 `attack.py`；封装脚本固定了 `--split val`、`--n_inputs 100`（以各脚本实际为准）、`--cache_dir ./models_cache` 等，**额外 CLI 从第 3 个位置参数之后传入**。

### 3.1 脚本通用位置参数

| 参数 | 含义 |
|------|------|
| 第 1 个 | **DATASET**：`cola`、`sst2`、`rotten_tomatoes` 等 |
| 第 2 个 | **BATCH_SIZE**：批大小 |
| 其后 | 任意传给 `attack.py` 的选项，例如 `--rank_tol 1e-7`、`--task next_token_pred` |

### 3.2 各模型 / 设置对应的命令

| 实验 | 命令 | 说明 |
|------|------|------|
| GPT-2 | `./scripts/gpt2.sh DATASET BATCH_SIZE [额外参数...]` | 默认 `model_path=gpt2`，序列分类 |
| GPT-2 Large | `./scripts/gpt2-large.sh DATASET BATCH_SIZE [额外参数...]` | 大模型 |
| GPT-2 微调 | `./scripts/gpt2-ft.sh DATASET BATCH_SIZE [额外参数...]` | 使用微调权重 |
| 下一词预测任务 | `./scripts/gpt2.sh DATASET BATCH_SIZE --task next_token_pred` | 改任务类型 |
| Frobenius / MSE 损失 | `./scripts/gpt2.sh DATASET BATCH_SIZE --loss mse` | 损失函数 |
| ReLU 隐藏激活 | `./scripts/gpt2.sh DATASET BATCH_SIZE --hidden_act relu` | 激活函数 |
| LLaMA-2 | `./scripts/llama.sh DATASET BATCH_SIZE [额外参数...]` | 需 HF 与 token |
| BERT（启发式全开） | `./scripts/bert.sh DATASET BATCH_SIZE [额外参数...]` | |
| BERT（启发式关） | `./scripts/bert.sh DATASET BATCH_SIZE --l1_filter all --l2_filter overlap` | |
| LoRA 微调设置 | `./scripts/lora.sh DATASET BATCH_SIZE [额外参数...]` | 默认 rank r=256 |

**LLaMA Rotten Tomatoes、batch 128（README 建议加速与数值稳定）：**

```bash
./scripts/llama.sh rotten_tomatoes 128 --rank_tol 1e-11 --l1_span_thresh 1e-4 --l2_span_thresh 5e-10
```

**ECHR 长序列：** 建议较小 `rank_tol` 与 `l1_span_thresh`（如 `1e-9`、`1e-4`），见 README。

## 4. FedAvg 设置（论文表 4）

默认数据集 `rotten_tomatoes`，`batch_size=16`，脚本内还固定了 `fedavg` 相关超参与部分 span / rank 选项。

**命令：**

```bash
./scripts/fed_avg.sh AVG_EPOCHS B_MINI LEARNING_RATE [额外参数...]
```

| 位置参数 | 含义 |
|----------|------|
| `AVG_EPOCHS` | FedAvg 外层迭代轮数（对应 `attack.py` 的 `--avg_epochs`） |
| `B_MINI` | 小批量大小（`--b_mini`） |
| `LEARNING_RATE` | 本地一步 SGD 学习率（`--avg_lr`） |
| 第 4 个起 | 额外传给 `attack.py` 的参数 |

## 6. 微调 GPT-2（含防御噪声训练）

需安装 `scikit-learn`（`pip install scikit-learn` 或 conda）。

**命令：**

```bash
python3 train.py --dataset DATASET --batch_size 32 --noise SIGMA --num_epochs NUM_EPOCHS \
  --save_every 100 --model_path MODEL
```

| 参数 | 含义 |
|------|------|
| `--dataset` | `cola`、`sst2`、`rotten_tomatoes` 等 |
| `--noise` / `SIGMA` | 高斯噪声强度；**无防御设为 `0.0`** |
| `--num_epochs` | 训练轮数 |
| `--batch_size` | 批大小（示例中为 32） |
| `--save_every` | 保存间隔（步或 epoch，以 `train.py` 实现为准） |
| `--model_path` | 基座模型名或路径 |

checkpoint 位于 `finetune/<TRAINING_METHOD>_<EPOCH>/`（`TRAINING_METHOD` 为 `lora` 或 `full`）。

## 7. 防御 Baselines 实验（`DEFENSE_BASELINES.md`）

### 7.1 一键批量：所有防御 + 强度扫参

**命令：**

```bash
bash scripts/defense_baselines.sh DATASET BATCH_SIZE MODEL_PATH N_INPUTS [传给 attack.py 的额外参数...]
```

| 位置参数 | 含义 |
|----------|------|
| `DATASET` | 如 `sst2`、`cola`、`rte`、`rotten_tomatoes` |
| `BATCH_SIZE` | 批大小；**MixUp 需 `batch_size >= 2` 才有真实混合**（为 1 时会退化为普通梯度） |
| `MODEL_PATH` | 如 `gpt2`、`bert-base-uncased`、`openai-community/gpt2-large` |
| `N_INPUTS` | 攻击/重建的样本条数 |
| 第 5 个起 | 额外传给 `attack.py` 的参数（如 Soteria 加速：`--defense_soteria_sample_dims 256`） |

**示例：**

```bash
bash scripts/defense_baselines.sh sst2 2 gpt2 20
bash scripts/defense_baselines.sh sst2 2 bert-base-uncased 3
```

脚本会对除 `none` 外的防御做**强度多档 sweep**（隐私–效用曲线），日志在 `log/runs/defense_baselines_*` 下，含各变体 `*.txt` 与 `summary.txt`（表中有 `variant`、`param`）。环境变量 `DAGER_LOG_DIR`、`DAGER_NO_AUTO_LOG` 同样适用。

**扫参对应关系（与 DEFENSE_BASELINES 一致）：**

| 防御 | 扫参 CLI | 典型取值方向 |
|------|-----------|----------------|
| `none` | — | 单次 |
| `noise` / `dpsgd` | `--defense_noise` | 弱→强如 `1e-6` … `1e-3` |
| `topk` | `--defense_topk_ratio` | `0.01` … `0.9` |
| `compression` | `--defense_n_bits` | `2` … `32` |
| `soteria` | `--defense_soteria_pruning_rate` | `10` … `90` |
| `mixup` | `--defense_mixup_alpha` | `0.1` … `2.0` |

### 7.2 单种防御：直接调用 `attack.py`

以下与批量脚本一致的核心选项：**数据集、划分、样本数、批大小、过滤器、模型、设备、任务、缓存目录、防御类型**。

**公共骨架（可按需改 `DATASET` / `n_inputs` / `batch_size`）：**

```text
python attack.py --dataset <DATASET> --split val --n_inputs <N> --batch_size <B> \
  --l1_filter all --l2_filter non-overlap --model_path <MODEL> \
  --device cuda --task seq_class --cache_dir ./models_cache \
  <防御相关参数>
```

| 实验 | `--defense` 与其它必/选参数 |
|------|-----------------------------|
| 无防御 | `--defense none` |
| 噪声注入 | `--defense noise --defense_noise <σ>`（必给噪声） |
| DP-SGD | `--defense dpsgd --defense_noise <σ>`；可选 `--defense_clip_norm`（默认 `1.0`） |
| Top-k 稀疏 | `--defense topk`；可选 `--defense_topk_ratio`（默认 `0.1`） |
| 梯度压缩 | `--defense compression`；可选 `--defense_n_bits`（默认 `8`） |
| Soteria | `--defense soteria`；可选 `--defense_soteria_pruning_rate`（默认 `60.0`）、`--defense_soteria_sample_dims`（大模型建议 `256` 等以加速） |
| MixUp | `--defense mixup`；可选 `--defense_mixup_alpha`（默认 `1.0`）；**仅 `seq_class` 有意义** |

### 7.3 防御相关 CLI 默认值一览

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `--defense` | `none` | 防御类型 |
| `--defense_noise` | 未设置 | `noise` / `dpsgd` 的高斯噪声标准差（这两类必须设置） |
| `--defense_clip_norm` | `1.0` | DP-SGD 堆叠梯度 L2 裁剪范数 |
| `--defense_topk_ratio` | `0.1` | 按 \|g\| 每个张量保留的比例（0.1 即 10%） |
| `--defense_n_bits` | `8` | 均匀量化位数 |
| `--defense_soteria_pruning_rate` | `60.0` | 低于该百分位得分的列被剪枝 |
| `--defense_soteria_sample_dims` | 未设置 | 若设置则随机采样该数量的隐维打分（加速） |
| `--defense_mixup_alpha` | `1.0` | MixUp 中 `Beta(alpha, alpha)` |

### 7.4 注意事项（防御）

- `noise` / `dpsgd` 必须提供 `--defense_noise`。
- `mixup` 在 `next_token_pred` 下会退回普通 `compute_grads`。
- `soteria` 在 `train_method=lora` 下未实现，会 `NotImplementedError`。
- Top-k / 压缩极弱时 ROUGE 长期为 0，可考虑放宽 `--l1_span_thresh` 或噪声梯度解码路径（见仓库内 `uses_noisy_gradient_decoding` 说明）。
